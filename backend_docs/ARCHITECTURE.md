# Архитектура проекта UniRide

## Текущая архитектура (реализованная)

Проект использует **гибридную архитектуру**, сочетающую клиентское приложение на React и бэкенд-сервер, развернутый через ngrok для демонстрационных целей.

### Компоненты системы:

1. **Клиентская часть (Frontend)**
   - React 18 + TypeScript приложение
   - Управление состоянием через React Hooks
   - Интеграция с 2GIS API для картографии
   - Локальное кэширование в LocalStorage

2. **Серверная часть (Backend)**
   - Развернута через ngrok для публичного доступа
   - Обработка API-запросов от клиента
   - Временное хранилище данных (в памяти или SQLite)
   - Аутентификация и авторизация

3. **Внешние сервисы**
   - 2GIS API для карт и геокодирования
   - Ngrok для туннелирования и публикации бэкенда

## Эволюция архитектуры

Первоначально проект планировался как полноценное Django-приложение, но для целей демонстрации и быстрого прототипирования была выбрана текущая гибридная архитектура, которая:
1. Позволяет быстро продемонстрировать работоспособность
2. Сохраняет возможность миграции на Django в будущем
3. Обеспечивает гибкость в разработке
